Use Amazon S3 when you want authentik to store files in a native AWS S3 bucket.

## Create the bucket in AWS

1. Log in to the [AWS Management Console](https://console.aws.amazon.com/).
2. Go to **Amazon S3** > **Buckets**.
3. Click **Create bucket**.
4. Enter `<bucket_name>` as the bucket name.
5. Select the AWS region that should store authentik files.
6. Keep **Block all public access** enabled.
7. Click **Create bucket**.

You can also create the bucket with the AWS CLI:

```bash
aws s3api create-bucket \
  --bucket=<bucket_name> \
  --region=<region> \
  --create-bucket-configuration LocationConstraint=<region>
```

For the `us-east-1` region, omit `--create-bucket-configuration`:

```bash
aws s3api create-bucket \
  --bucket=<bucket_name> \
  --region=us-east-1
```

## Create AWS credentials

If authentik runs outside AWS, create an IAM policy and access key for authentik:

1. In the AWS Console, go to **IAM** > **Policies**.
2. Click **Create policy**.
3. Click **JSON**.
4. Paste the policy from [Bucket permissions](./index.mdx#bucket-permissions).
5. Replace `<bucket_name>` with the bucket that you created.
6. Click **Next**.
7. Enter a policy name, for example `authentik-s3-storage`.
8. Click **Create policy**.
9. Go to **IAM** > **Users**.
10. Click **Create user**.
11. Enter a user name, for example `authentik-s3`.
12. Click **Next**.
13. Select **Attach policies directly**.
14. Select the policy that you created.
15. Click **Next**, then click **Create user**.
16. Open the new user and go to **Security credentials**.
17. Click **Create access key**.
18. Select the use case that matches your deployment, then click **Next**.
19. Click **Create access key**.
20. Copy the access key ID and secret access key.

## Configure authentik

Add the shared S3 settings, then set the AWS region:

```env
AUTHENTIK_STORAGE__S3__REGION=<region>
```

Do not set `AUTHENTIK_STORAGE__S3__ENDPOINT` for Amazon S3.

If authentik runs on AWS with an instance role, task role, web identity, or AWS profile, you can omit `AUTHENTIK_STORAGE__S3__ACCESS_KEY` and `AUTHENTIK_STORAGE__S3__SECRET_KEY`.
