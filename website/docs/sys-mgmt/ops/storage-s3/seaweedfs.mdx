## Configure the SeaweedFS S3 service

Configure and expose the SeaweedFS S3 service before you configure authentik:

1. Log in to the host or cluster where SeaweedFS is deployed.
2. Enable the SeaweedFS S3 service for your deployment.
3. Create or configure an S3 identity for authentik.
4. Grant the identity permission to list the bucket and to read, write, and delete objects.
5. Start or reload the SeaweedFS S3 service with the updated S3 configuration.
6. Create the bucket that authentik should use, for example with the AWS CLI:

    ```bash
    AWS_ACCESS_KEY_ID=<access_key> AWS_SECRET_ACCESS_KEY=<secret_key> aws s3api \
      --endpoint-url=https://s3.seaweedfs.company \
      create-bucket \
      --bucket=<bucket_name>
    ```

7. Configure CORS with the policy from [CORS policy](./index.mdx#cors-policy).

The endpoint depends on your SeaweedFS deployment. For example:

```text
https://s3.seaweedfs.company
```

## Configure authentik

Add the shared S3 settings, then set the SeaweedFS endpoint:

```env
AUTHENTIK_STORAGE__S3__ENDPOINT=https://s3.seaweedfs.company
AUTHENTIK_STORAGE__S3__ADDRESSING_STYLE=path
```
