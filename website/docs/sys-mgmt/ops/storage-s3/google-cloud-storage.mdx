## Create the bucket in Google Cloud

1. Log in to the [Google Cloud console](https://console.cloud.google.com/).
2. Select the project that should own the bucket.
3. Go to **Cloud Storage** > **Buckets**.
4. Click **Create**.
5. Enter `<bucket_name>` as the bucket name.
6. Select the location and storage settings for your deployment.
7. Click **Create**.

## Create Google Cloud credentials

1. In the Google Cloud console, go to **IAM & Admin** > **Service Accounts**.
2. Click **Create service account**.
3. Enter a service account name, for example `authentik-s3`.
4. Click **Create and continue**.
5. Grant the service account the **Storage Object Admin** role on the bucket.
6. Click **Done**.
7. Go to **Cloud Storage** > **Settings**.
8. Open the **Interoperability** tab.
9. Click **Create a key for a service account**.
10. Select the service account that you created.
11. Click **Create key**.
12. Copy the access ID and secret.

Google Cloud Storage uses the following interoperability endpoint:

```text
https://storage.googleapis.com
```

## Configure authentik

Add the shared S3 settings, then set the Google Cloud Storage endpoint:

```env
AUTHENTIK_STORAGE__S3__ENDPOINT=https://storage.googleapis.com
AUTHENTIK_STORAGE__S3__ADDRESSING_STYLE=path
```

Use the HMAC access ID as `AUTHENTIK_STORAGE__S3__ACCESS_KEY` and the HMAC secret as `AUTHENTIK_STORAGE__S3__SECRET_KEY`.
