You can require a user to change their password the next time that they log in with a password. This is useful after an administrator sets a temporary password or when a user must rotate their current password before they continue.

The requirement uses the password stage's configured password change flow. In the default authentication flow, the default password stage uses the `default-password-change` flow.

:::info
This requirement applies when the user's authentication path includes a password stage. If a custom authentication flow uses a password stage, ensure that the stage has **Configure flow** set to a password change flow.
:::

## Require a password change when resetting a password

1. Log in to authentik as an administrator and open the authentik Admin interface.
2. Navigate to **Directory** > **Users**.
3. Click the name of the user whose password you want to reset.
4. On the **Overview** tab, click **Reset password**.
5. Enter the new password.
6. Enable **Require password change on next login**.
7. Click **Update**.

The user can log in with the password that you set. Before the login can continue, authentik redirects them to the configured password change flow.

## Require a password change for the current password

You can also require a password change without setting a new password for the user.

1. Log in to authentik as an administrator and open the authentik Admin interface.
2. Navigate to **Directory** > **Users**.
3. Click the edit icon beside the user.
4. Enable **Require password change on next login**.
5. Click **Update**.

The next time that the user logs in with their current password, authentik redirects them to the configured password change flow.

## Clear the password change requirement

authentik clears the password change requirement after the user changes their password.

An administrator can also clear the requirement by editing the user and disabling **Require password change on next login**, or by resetting the user's password without enabling **Require password change on next login**.
