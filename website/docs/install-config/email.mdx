Use this page to configure SMTP for authentik and verify that email delivery is working.

Global email settings are used for administrator notifications, release and configuration alerts, [notification rules](../sys-mgmt/events/notifications.md), and any [Email stage](../../add-secure-apps/flows-stages/stages/email/) configured to use global settings.

:::warning
Some hosting providers block outgoing SMTP ports, in which case you will need to host an SMTP relay on a different port with a different provider.
:::

## Before you begin

Have the following values ready:

- SMTP host
- SMTP port
- Optional username and password
- The sender address for `AUTHENTIK_EMAIL__FROM`
- The TLS mode required by your provider

## Choose the TLS mode

authentik supports the two SMTP TLS modes commonly documented by mail providers:

| Provider says           | Port          | `AUTHENTIK_EMAIL__USE_TLS` | `AUTHENTIK_EMAIL__USE_SSL` |
| ----------------------- | ------------- | -------------------------- | -------------------------- |
| `STARTTLS`              | Usually `587` | `true`                     | `false`                    |
| `SSL` or `Implicit TLS` | Usually `465` | `false`                    | `true`                     |
| No TLS                  | Often `25`    | `false`                    | `false`                    |

Do not enable both settings at the same time.

If your provider documentation says "use STARTTLS", set `use_tls` to `true` and leave `use_ssl` disabled.

## Configure global email settings

Set the global SMTP configuration in your deployment, then redeploy authentik.

<Tabs
  groupId="deployment"
  defaultValue="docker"
  values={[
    { label: "Docker", value: "docker" },
    { label: "Kubernetes", value: "kubernetes" },
  ]}
>
  <TabItem value="docker">

Add the following block to your `.env` file:

```sh
# SMTP server
AUTHENTIK_EMAIL__HOST=localhost
AUTHENTIK_EMAIL__PORT=25
# Optional SMTP authentication
AUTHENTIK_EMAIL__USERNAME=
AUTHENTIK_EMAIL__PASSWORD=
# STARTTLS / explicit TLS, usually on port 587
AUTHENTIK_EMAIL__USE_TLS=false
# Implicit TLS, usually on port 465
AUTHENTIK_EMAIL__USE_SSL=false
AUTHENTIK_EMAIL__TIMEOUT=10
# Sender address, for example authentik@example.com
AUTHENTIK_EMAIL__FROM=authentik@localhost
```

  </TabItem>
  <TabItem value="kubernetes">

Add the following block under `authentik:` in your `values.yaml` file:

```yaml
email:
    # -- SMTP server
    host: ""
    port: 25
    # -- Optional SMTP authentication. Leave empty to disable authentication.
    username: ""
    # -- Optional SMTP authentication. Leave empty to disable authentication.
    password: ""
    # -- STARTTLS / explicit TLS, usually on port 587.
    use_tls: false
    # -- Implicit TLS, usually on port 465.
    use_ssl: false
    # -- Connection timeout in seconds
    timeout: 10
    # -- Email sender, for example "authentik <authentik@example.com>"
    from: "authentik@localhost"
```

  </TabItem>
</Tabs>

### When to use stage-specific settings

[Email stages](../../add-secure-apps/flows-stages/stages/email/) can either:

- use the global SMTP settings described above, or
- use their own stage-specific SMTP host, port, credentials, and TLS settings

Use stage-specific settings only when a flow must send through a different mail server than the rest of authentik.

## Test email delivery

After configuring SMTP, send a test message from the authentik server:

```shell
ak test_email <to_address>
```

To test a specific Email stage instead of the global settings, include `-S`:

```shell
ak test_email <to_address> [-S <stage_name>]
```

If you omit `-S`, authentik uses the global SMTP configuration.

## Example: Google Workspace SMTP relay

One reliable way to send email through Google is [Google's SMTP relay service](https://support.google.com/a/answer/2956491). Google also documents the broader setup flow in [Send email from a printer, scanner, or app](https://support.google.com/a/answer/176600?hl=en).

First, determine the outbound IP address used by authentik and allow it in the Google Workspace **SMTP relay service** settings. Then configure the relay with these options:

- Set **Allowed Senders** to `Only addresses in my domains`.
- Set **Authentication** to `Only accept mail from the specified IP addresses`.
- Do not set **Require SMTP Authentication**.
- Select **Require TLS encryption**.

<Tabs
  groupId="deployment"
  defaultValue="docker"
  values={[
    { label: "Docker", value: "docker" },
    { label: "Kubernetes", value: "kubernetes" },

  <TabItem value="docker">
Set the following environment variables:

```sh
AUTHENTIK_EMAIL__HOST=smtp-relay.gmail.com
AUTHENTIK_EMAIL__PORT=587
AUTHENTIK_EMAIL__USE_TLS=true
AUTHENTIK_EMAIL__USE_SSL=false
AUTHENTIK_EMAIL__TIMEOUT=30
```

Redeploy authentik, then verify delivery with `ak test_email`.

  </TabItem>
  <TabItem value="kubernetes">

Set the following values in the `email` section:

```yaml
email:
    host: "smtp-relay.gmail.com"
    port: 587
    use_tls: true
    use_ssl: false
    timeout: 30
```

Redeploy authentik, then verify delivery with `ak test_email`.

  </TabItem>
</Tabs>

## Trust a custom SMTP certificate authority

If your SMTP server uses a certificate signed by a private or internal certificate authority, mount that CA certificate into the authentik `server` and `worker` containers and point `SSL_CERT_FILE` at it.

<Tabs
  groupId="deployment"
  defaultValue="docker"
  values={[
    { label: "Docker", value: "docker" },
    { label: "Kubernetes", value: "kubernetes" },
  ]}
>
  <TabItem value="docker">

1. Add the certificate mount and environment variable to both the `server` and `worker` services in your Docker Compose file:

```yaml
volumes:
    - /path/to/<cert_name>.crt:/etc/ssl/certs/<cert_name>.crt:ro
environment:
    - SSL_CERT_FILE="/etc/ssl/certs/<cert_name>.crt"
```

2. Redeploy the containers.

  </TabItem>
  <TabItem value="kubernetes">

1. Create a `ConfigMap` containing the CA certificate:

```sh
kubectl create configmap my-custom-cert --from-file=<cert_name>.crt -n <your_namespace>
```

2. Add a volume in your `values.yaml` file:

```yaml
volumes:
    - name: custom-ca
      configMap: # or use secret if preferred
          name: my-custom-cert
```

3. Add a `volumeMount` and `SSL_CERT_FILE` to both the `server` and `worker` containers:

```yaml
volumeMounts:
    - name: custom-ca
      mountPath: /etc/ssl/certs/<cert_name>.crt
      subPath: <cert_name>.crt
      readOnly: true
env:
    - name: SSL_CERT_FILE
      value: /etc/ssl/certs/<cert_name>.crt
```

4. Recreate the pods.

  </TabItem>
</Tabs>
