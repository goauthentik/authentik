## Highlights

- **User security notifications**: New optional email notifications keep users informed about security-relevant changes to their own account: password changes, MFA device changes, blocked logins, account lockdown, and a welcome email for newly created accounts.

## Breaking changes

### `static_authenticator_disable` event removed

authentik no longer creates the `static_authenticator_disable` event. Removing any MFA device, including static authenticators, now creates the generic `mfa_device_removed` event. Update any Event Matcher policies or notification rules that reference the removed action.

## New features and improvements

### User security notifications

authentik now ships notification rules that email users about security-relevant changes to their own account: interactive password changes, MFA devices being added or removed, logins blocked because of impossible travel, [account lockdown](../../security/account-lockdown.md), and a welcome email when their account is created. The rules are disabled by default; to learn how to enable and customize them, refer to the [user security notifications documentation](../../sys-mgmt/events/user-security-notifications.md).

The feature is built on new primitives that are also available to custom notification rules:

- New event types — `user_created`, `login_blocked`, `mfa_device_added`, and `mfa_device_removed` — with context fields, such as the origin of a change, that notification policies can match on.
- Notification rules can send the notification to the user affected by an event (who is not necessarily the user that triggered it), and can be disabled without removing their configuration.
- Email notification transports can render the email subject from a template.

## Upgrading

This release does not introduce any new requirements. You can follow the upgrade instructions below; for more detailed information about upgrading authentik, refer to our [Upgrade documentation](../../install-config/upgrade.mdx).

:::warning
When you upgrade, be aware that the version of the authentik instance and of any outposts must be the same. We recommend that you always upgrade any outposts at the same time you upgrade your authentik instance.
:::

### Docker Compose

To upgrade, download the new docker-compose file and update the Docker stack with the new version, using these commands:

```shell
wget -O docker-compose.yml https://goauthentik.io/version/xxxx.x/lifecycle/container/compose.yml
docker compose up -d
```

The `-O` flag retains the downloaded file's name, overwriting any existing local file with the same name.

### Kubernetes

Upgrade the Helm Chart to the new version, using the following commands:

```shell
helm repo update
helm upgrade authentik authentik/authentik -f values.yaml --version ^xxxx.x
```
