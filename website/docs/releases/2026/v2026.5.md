## Breaking changes

### Listening on multiple IPs

For advanced use cases, authentik now supports setting listening settings to a comma-separated list of IPs. With this change, the default IP we listen on changed from `0.0.0.0` to `[::]` to better match ecosystem standards. Some IPv4-only environments might need to adapt those settings.

## New features and improvements

### Account Lockdown: :ak-enterprise

Account Lockdown allows you to immediately secure accounts that you suspect have been compromised. You can now do any combination of:

- defining a flow to lock a user
- deleting tokens
- disabling passwords
- terminating sessions

This acts as a "panic button" that can be pressed by an Admin or the owner of an account in case an account is thought to be compromised.

### General Improvements

### Command Palette

Command palette allows you to use CMD + K to search the authentik UI for different features and things you'd like to customize.

### Fleet Conditional Access: :ak-enterprise

authentik can now verify user devices based on their Fleet certificates, without the authentik agent. This is achieved with the Fleet Connector and an mTLS stage.

Link to docs: website/docs/endpoint-devices/device-compliance/fleet-conditional-access.md

### WebAuthn Client Hints support

https://github.com/w3c/webauthn/blob/main/explainers/hints-for-webauthn-clients.md
The WebAuthn Stage now supports the `hints` parameter from the WebAuthn Level 3 spec. Admins can configure one or more hints (`security-key`, `client-device`, or `hybrid`) to tell the browser which authenticator type to expect. The browser uses this to skip straight to the relevant selection UI during passkey registration and authentication, reducing friction especially in enterprise deployments where security keys are mandatory.

Keep in mind that hints are advisory — they only affect the browser UI, not policy. Authenticator type requirements still need to be enforced server-side.

### AKQL is OSS now

lean towards using first: https://docs.goauthentik.io/sys-mgmt/events/logging-events/#advanced-queries - https://docs.goauthentik.io/users-sources/user/user_basic_operations/#advanced-queries
The authentik query language tool - AKQL - was an enterprise only feature that allowed for querying logs and users. This allows you to search for events based on specific attributes, for example:
context.geo.country = "Germany". We have open-sourced AKQL, meaning it is now free for everyone to use.

### Generally improved UI and accessibility

Wizards throughout authentik have been reworked, including a new user creation wizard, an improved binding wizard and a new invitation wizard. The invitation wizard in particular now makes it easy for administrators to send invites to new users. It guides admins through the process of configuring an invite system and sending the invites to users. Service accounts are now created in the same location as all other users. Every button in authentik now has a screen reader friendly label that tells the user the full action that will execute when the button is pressed. These changes have all been reflected in the docs.

**note** many accessibility changes that will be added by teffen

### Small general improvements (SAML Issuer, Hide apps)

SAML Issuer - We now automatically generate your SAML issuer url. You may still overwrite the default SAML Issuer.
Hide Apps - You can hide apps from the Application Dashboard for situations when a user needs access to an application that doesn't need to show up in the Application Dashboard.

### Performance improvements

We've reworked a significant section of our backend in Rust. This had lead to an approximately 30% drop in memory usage. If you're a developer, check the updated Developer Docs to install Rust.

There are also improvements to PostgreSQL connection handling, _more things that Marc and Jens can list_.

The Admin Interface when loaded in a browser is less resource intensive due to a lazy loading improvement with modals.

## Upgrading

This release does not introduce any new requirements. You can follow the upgrade instructions below; for more detailed information about upgrading authentik, refer to our [Upgrade documentation](../../install-config/upgrade.mdx).

:::warning
When you upgrade, be aware that the version of the authentik instance and of any outposts must be the same. We recommend that you always upgrade any outposts at the same time you upgrade your authentik instance.
:::

### Docker Compose

To upgrade, download the new docker-compose file and update the Docker stack with the new version, using these commands:

```shell
wget -O docker-compose.yml https://goauthentik.io/version/2026.5/lifecycle/container/compose.yml
docker compose up -d
```

The `-O` flag retains the downloaded file's name, overwriting any existing local file with the same name.

### Kubernetes

Upgrade the Helm Chart to the new version, using the following commands:

```shell
helm repo update
helm upgrade authentik authentik/authentik -f values.yaml --version ^2026.5
```
